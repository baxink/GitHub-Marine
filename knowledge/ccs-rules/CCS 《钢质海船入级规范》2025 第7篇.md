# 第1章 通 则

# 第1节 一般规定

# 1.1.1 一般要求

1.1.1.1 本篇适用于安装在海船上的机电设备和装置的自动化系统。

1.1.1.2 本篇所述自动化系统包括控制系统、安全系统和报警系统（包括显示）。

1.1.1.3 对于满足本篇要求的不同自动化等级的船舶，可授予下列附加标志：

（1）推进装置由驾驶室控制站遥控，机器处所包括机舱集控站（室）周期无人值班——AUT-0；

（2）机舱集控站（室）有人值班对机电设备进行监控——MCC；

（3）推进装置由驾驶室控制站遥控，机器处所有人值班——BRC。

1.1.1.4 设有符合本篇要求的自动化系统的船舶，均应能适应所有航海工况（包括机动操纵航行）。

1.1.1.5 设有符合本篇要求的自动化系统的船舶的安全性，应与机电设备有人直接看管的船舶相同，并应有措施保证在自动化系统失效时能在机旁对机电设备进行有效的人工操作。

# 1.1.2 定义

1.1.2.1 本篇有关定义如下：

（1）自动控制：系指具有自动调节特性，无需操作人员参与即可按预定指令对设备进行操作的控制。

（2）遥控：系指由操作人员通过机械、电气、电子、气动、液压、电磁（无线电）及光学方式或其组合方式远距离对设备进行操作的控制。

（3）就地控制：系指位于机电设备近旁由操作人员对设备进行直接的人工操作的控制。

（4）控制站（室）：系指具有监视功能且能够对机电设备实施控制的处所。在本篇中的控制站（室）主要有下列4类：

① 机舱集控站（室）：系指机舱内自动化设备的所有监控设施集中布置的控制站（室）；

② 驾驶室控制站：系指设在驾驶室内对推进装置及其他设备进行监控的控制站；

(3) 就地控制站：系指对机电设备实施就地控制的控制站；

④ 其他控制站：系指船上除上述3种控制站以外的控制站。

（5）安全系统：系指当发生危及主推进装置、锅炉、电站以及其他重要机电设备的严重故障时，能使发生故障的机电设备，按下列3种类型自动产生保护性动作的系统：

a类：立即停止运行，如主柴油机紧急停车，锅炉紧急停炉，紧急切断用电设备电源等，而且非经人工复位，该设备不应再投入运行；

b类：暂时调节到可以勉强运行的状态，如降低功率或转速等；

c类：起动和投入备用设备，以恢复正常的运行状态。

（6）报警：系指当被监控的机电设备或系统超出预定参数范围时所发出的听觉和视觉信号，并能识别机器处所内具体的故障情况和位置。

(7) 组合报警：系指被监控的机电设备或系统处于任何非正常状况下发出的任一报警，通过一个报警通道所送出的公用报警。

(8) 故障安全原则: 系指当一个元件或一个系统出现故障或误动作时, 该元件或系统的输出能自动处于预定的安全状态。

（9）越控：系指越过控制过程中的某一程序或某一安全保护功能，在短时间内强制机电设备继续运行以保证船舶安全的特殊控制措施。

（10）紧急停止装置：系指独立于控制系统，在紧急情况下用人工停止机电设备运行的装置。如主柴油机紧急停车按钮、锅炉紧急停炉按钮、用电设备电源紧急切断按钮等。

（11）冗余设计：系指当系统或设备发生故障时所采用的备用技术手段，依靠其能即刻参与相应的功能而使动作状态持续运行，或者通过恢复原有功能而使运行状态得以延续。

（12）节点：系指数据通信联络的内部连接点。

（13）网络：系指计算机之间进行数据传递和交换的通信网。

（14）重大改装：系指船上的计算机系统除外围设备外，发生下列之一或组合的情况导致功能和安全性能发生实质性改变：

①计算机硬件配置的改变；

②软件升级；

(3)网络（包括拓扑结构）的更改。

# 1.1.3 图纸资料

1.1.3.1 具有自动化附加标志的船舶，应将其相应的本篇所涉及设备的控制、报警（显示）和安全系统的下列图纸资料提交批准：

（1）测量显示点；

（2）报警点（包括控制站、室内报警信号显示的位置、报警方式）；

（3）安全系统的项目；

（4）自动化系统的动力源（包括电力、气动、液压）系统图。

1.1.3.2 如有必要，还应提交CCS要求送审的其他图纸资料。

# 1.1.4 产品证件

1.1.4.1 用于船舶自动化系统的设备和装置等的持证要求详见本规范第1篇第3章附录1《船舶入级产品持证要求一览表》。

# 1.1.5 试验

1.1.5.1 自动化系统应连同被监控的机、电设备一起，按经CCS批准的试验大纲进行系泊和航行试验，以查明整个系统是否正常工作。

1.1.5.2 应记录经试验满足本规范要求所确定的自动化系统报警点和安全系统参数的整定值，并保存在船上备查。

# 第2章 基本要求

# 第1节 一般规定

# 2.1.1 环境条件

2.1.1.4 自动化系统应能在各方向倾斜及摇摆  $22.5^{\circ}$  （周期10s）以及垂直方向线性加速度  $\pm 9.8\mathrm{m / s}^2$  时正常工作。

2.1.1.5 自动化系统应能适应船上盐雾、油雾、霉菌及灰尘等环境。

# 2.1.2 其他工作条件

2.1.2.1 自动化系统应能满足本规范第4篇第1章第2节第1.2.2和1.2.3的要求。

2.1.2.2 自动化系统应具有必要的电磁兼容性。

2.1.2.3 自动化系统的液压和气动设备，应能在动力源压力变化额定值的  $\pm 20\%$  时正常工作，在设计压力值的1.5倍时不损坏。

# 2.1.3 故障安全

2.1.3.1 控制系统、报警系统和安全系统应按故障安全原则设计。故障安全原则不仅应考虑自动化系统本身以及与之有关的机械，而且应考虑全部机械装置，以及船舶和人员的安全。

2.1.3.2 控制系统、报警系统和安全系统的设计，应能使运行过程中出现的一个故障不会导致其他故障的产生，并且其产生的危险性降到尽可能低的程度。

2.1.3.3 自动控制和遥控系统应确保持续、有效、可靠地运行。

# 2.1.4 独立性

2.1.4.1 控制系统、安全系统和报警系统应设计成或使其功能相互独立，当其中的某一或多个系统发生故障时，应不影响其他系统的正常工作。

2.1.4.2 在任何情况下，a类安全系统应独立于控制系统和报警系统，使之不受其他系统故障的影响；但b、c类安全系统不要求与其他的控制系统和报警系统完全独立。

2.1.4.3 机电设备不同单元的安全系统也应各自独立。当其中某一部分的安全系统发生故障时应不妨碍另一部分安全系统的正常工作。

# 2.1.5 检测和锁定

2.1.5.1 对控制系统、安全系统和报警系统的功能应能进行检查。如控制设备、安全设备和报警设备的灵敏度和极限值等整定值可以调节，则调定的数值应易于检查、辨认并予以锁定。

# 2.1.6 供电

2.1.6.1 安全系统、报警系统以及正常供电失电时仍有必要使用的控制系统（例如电站的自动化系统），应能在正常供电失电时自动转接到备用电源。该备用电源可以采用蓄电池组，其容量应至少能维持  $30\mathrm{min}$  供电的需要。

若上述系统可能因电源的中断而受到有害影响时，则应采用不中断的方式转换到备用电源。

2.1.6.2 当安全系统、报警系统和控制系统的电源故障①时应发出视觉和听觉报警。

2.1.6.3 控制系统、安全系统和报警系统应由独立的最后分路供电，并设有短路和过载保护。

# 第2节 控制系统

# 2.2.1 一般要求

2.2.1.1 控制系统包括自动控制系统和遥控系统。

2.2.1.2 控制系统应具有良好的控制品质。控制系统以及被控制的机电设备，在其全部运行工况范围内应具有满意的特性，即应能稳定工作，不会使机电设备受到有害的机械负荷和热负荷，并具有必要的控制精度。

2.2.1.3 控制系统的设计，应能使控制系统中出现的故障对控制过程产生的危险性降到尽可能低的程度，并且不会使备用的自动和/或手动控制失效。

2.2.1.4 对于安全操纵船舶所必需的所有机电设备，当自动控制系统或遥控系统发生故障或失效（包括动力源中断）时，应能：

（1）发出报警信号；

（2）及时更换损坏的部件或投入备用设备以恢复正常的控制功能（如适用时）；

（3）转换到就地手动控制，转换控制时不应引起机电设备运行状态的严重变化。

2.2.1.5 除非操作上必需，主推进装置（包括主机和调距桨）、锅炉和电站的控制系统应各自独立，每一独立推进装置应设有独立的控制系统。

2.2.1.6 如有措施能保证本章 2.1.4.1 所述的各系统共用部分的设备能受到有效的监测，且在发生故障时能发出报警，可允许其局部共用（见本章 2.7.1.4）。

# 2.2.2 液压和气动动力源

2.2.2.1 控制系统的液压动力源及管系，应满足本规范第3篇第2章和第4章第7节的有关要求以及下列要求：

(1) 当运行泵的压力低于预定值时, 备用液压泵应能自动起动投入工作, 并应发出报警;

（2）当系统的压力低于正常工作压力预定值时，应发出报警。

2.2.2.2 控制系统的气动动力源及管系，应满足本规范第3篇第2章的有关要求以及下列要求：

（1）气动控制用空气瓶可以由主机起动空气瓶或专用空气压缩机供气；

(2) 当气动控制用空气瓶由专用空气压缩机供气时, 应设置两台空气压缩机, 其中一台为备用, 或者设一台专用空气压缩机和由主空气瓶供气, 当气压低于要求值时, 空气压缩机应自动起动并投入工作;

（3）气动控制的供气管路中应设置安全阀。安全阀的开启压力应不大于额定工作压力的1.1倍；

(4) 供气管路中应设置减压阀、滤器、油水分离器和干燥器, 以使空气保持干燥, 不

含杂质和油分。一般应至少各设置两只减压阀、滤器和干燥器，并应并联装设，如有措施能保证迅速完成上述器件的保养、更换工作而又不影响气动控制系统的正常工作，则允许各设置一只滤器和干燥器。

# 第3节 安全系统

# 2.3.1 一般要求

2.3.1.1 当发生危及主推进装置、锅炉、电站以及其他重要机电设备的严重故障时，安全系统应能按本篇1.1.2.1（5）中所述3种类型自动产生保护性动作，并应发出报警。

2.3.1.2 如因安全系统的动作而导致机电设备停止运行，则非经人工复位，该设备不应回自动投入运行。

2.3.1.3 为了避免机电设备发生不必要的运行中断，安全系统应在报警系统动作后随即以下列方式动作，使其产生的危害性尽可能低：

（1）起动和投入备用设备；

（2）降低负载或停止运行。

2.3.1.4 在安全系统动作后，应在控制站给出听觉和视觉报警，以便指示安全系统动作的原因。

# 2.3.2 越控

2.3.2.1 为保证船舶的安全，船舶的主机（如主柴油机、主汽轮机、主燃气轮机和电力推进装置等）一般应设有越控功能。

2.3.2.2 实施越控时，应予以指示并发出报警。

2.3.2.3 应有措施防止越控功能被误触动。

# 2.3.3 气动动力源及管系

2.3.3.1 安全系统的气动动力源及管系应满足本章2.2.2的要求，而且管路应尽实际可能与控制系统分开。

# 第4节 报警系统（包括显示）

# 2.4.1 一般要求

2.4.1.1 对被监控的机电设备和监控系统本身的所有故障应在有关的控制站（室）显示，并发出报警信号，以使值班轮机员知道发生的故障。

2.4.1.2 所有的报警应同时发出视觉和听觉报警信号。严重故障的报警光色一般采用红色，普通故障一般采用黄色。听觉信号应有足够的响度，并应与火警、电话及其他听觉信号（如释放  $\mathrm{CO}_{2}$  ）有明显的区别。

2.4.1.3 报警系统应能对同时发生的所有故障发出报警信号。对某个故障的报警和/或对报警的应答不应妨碍对其他同时发生的故障的报警和/或应答。

2.4.1.4 报警设备可设有消除听觉报警信号（消声）的装置，当报警信号应答后，按下该装置的消声按钮，消声时不应熄灭视觉报警信号但可改变视觉信号（如闪光转为平光等），该视觉信号仍应清晰可辨。在驾驶室和起居处所对听觉报警信号的局部消除不应中止机器处所的听觉报警。且机器处所听觉报警信号的消声按钮只允许设在机器处所或集控室内，同时视觉报警信号应一直保留到故障消除为止。故障消除后，该报警通道应能自动恢复到正常工作状态。

2.4.1.5 如单个报警在就地控制站已有显示，则也应在其他相关的控制站显示报警信号，但可采用组合报警的方式显示。

# 2.4.2 报警系统的检测与自检

2.4.2.1 报警系统应具有自检功能，即对自身的故障自动进行检测和报警（或指示）。自检的范围及程度可与维修更换的措施配合考虑。

2.4.2.2 报警系统应能对某些过程中无意义的报警信号进行闭锁。当闭锁由人工实施

时，应予以指示。

# 2.4.3 显示

2.4.3.1 可采用仪表、显示器等进行参数显示。参数可以单独显示，也可以选择显示；可用文字显示，也可以用图形显示；但显示均应清晰明了。指示灯信号的光色一般采用绿色或白色。与计算机配用的显示器及显示要求应满足IEC60092-504《船舶电气设施-第504部分：专辑一自动化、控制装置和仪器仪表》的要求。

# 第5节 控制站（室）

# 2.5.1 控制站（室）的布置

2.5.1.1 机舱集控站（室）一般应设在机器处所内。

2.5.1.2 机舱集控站（室）应尽可能设在振动和噪声较小的地方。

2.5.1.3 机器处所内的机舱集控站（室）应设计成具有隔声性能，窗玻璃应采用防碎型，并设有两个尽可能远离的进出口通道，其中之一应尽可能接近机器处所的脱险通道或者设有专用的脱险通道。

2.5.1.4 控制站（室）内监控设备、信号显示位置、操作手柄、开关、按钮等的布置应考虑人体工程学的原则，以利于操作、监视、维护以及人员的安全。

2.5.1.5 就地控制站应满足本篇 1.1.1.5 及 2.5.2 的有关要求。当其他控制站或者推进装置的自动或遥控系统任何部分出现故障或失效时，不应妨碍就地控制站对主推进装置就地控制的功能。

# 2.5.2 控制转换

2.5.2.1 各控制站（室）之间应能在其共同控制的机电设备正常运行或发生故障时实施控制转换。在进行这种转换时，不应引起机电设备运行状态的严重变化。

2.5.2.2 机舱集控站（室）与驾驶室控制站之间的控制转换，仅能在机舱集控站（室）进行；就地控制站与机舱集控站（室）或驾驶室控制站之间的控制转换，仅能在就地控制站进行。

2.5.2.3 控制转换应在得到应答后，方可进行。

2.5.2.4 在所有控制站（室）均应指示出哪个控制站（室）正在进行控制。

2.5.2.5 可由两个以上控制站（室）进行控制的机电设备，同一时刻仅能由一个控制站（室）控制。在各控制站（室）对主推进装置的控制器可以互联控制位置。

2.5.2.6 来自驾驶室的对主推进装置的指令，应在该装置的所有控制位置指示出来。

2.5.2.7 如从机舱集控站（室）通过直接的机械连接机构对主推进装置或其他机电设备进行控制，则可免设其相应的就地控制站。

# 2.5.3 其他控制站

2.5.3.1 如船上设有其他控制站涉及到机电设备及船舶的安全时，应提供相关的技术资料（如布置图、监控功能说明等），并应满足本篇的相关要求。

# 第6节 计算机系统

# 2.6.1 一般要求

2.6.1.1 本节适用于依靠软件实现其功能的计算机系统的设计、制造、调试和维护。本节适用于提供符合入级要求的控制、报警、监测、安全或内部通信功能的计算机系统。本节不适用于法定要求的计算机系统，例如装载仪/稳性计算机以及SOLAS公约第IV章和第V章规定的无线电通信设备和航行设备。

2.6.1.2 除本节的要求外，计算机系统还应满足CCS《电气电子产品型式认可试验指南》相关要求。在国际航行的所有客船和500总吨及以上货船上安装的计算机系统，应至少满足

《船舶网络安全指南》Cyber Security（P[SL0]）相关要求。

2.6.1.3 在开发计算机系统的硬件和软件时，可参考以下标准，以及CCS接受的其他标准：

（1）IEC61508《电气/电子/可编程电子安全相关系统的功能安全》；

（2）ISO/IEC/IEEE12207《系统和软件工程—软件生命周期过程》；

（3）ISO9001《质量管理体系要求》；

（4）ISO/IEC/IEEE90003《软件工程—计算机软件ISO9001：2015应用导则》；

（5）IEC60092-504《船舶电气设施—第504部分：专辑—自动化控制装置和仪器仪表》；

（6）ISO/IEC25000《系统和软件工程—系统和软件质量要求和评估（SQuaRE）-SQuaRE导则》；

（7）ISO/IEC 25041《系统和软件工程—系统和软件质量要求和评估（SQuaRE）-开发者、受让人和独立评价者用评价指南》；

（8）IEC61511《功能安全—安全仪表系统在过程工业中的应用》；

（9）ISO/IEC/IEEE15288《系统和软件工程—系统生命周期过程》；

（10）ISO 10007 《质量管理-配置管理指南》；

（11）ISO24060《船舶和海洋技术-船舶操作技术软件记录系统》。

# 2.6.2 定义

2.6.2.1 就本节而言，计算机系统有关的定义如下：

（1）黑盒描述：从系统外部观察到的关于系统功能、行为和性能的描述。

（2）黑盒测试方法：仅通过操纵输入和观察输出来验证系统、子系统或元件的功能、性能和鲁棒性。无需对系统内部运作有任何了解，仅通过关注被测系统/元件的可观察行为，以达到所需的验证水平。

（3）计算机系统：一种可编程电子设备，或一组可互操作的可编程电子设备，为达到一个或多个特定目的而组织起来，如收集、处理、维护、使用、共享、传播或处置信息。船载计算机系统包括IT和OT系统。计算机系统可以是通过网络连接的子系统的组合。船载计算机系统可以直接或通过公共通信方式（如互联网）与岸上的计算机系统、其他船舶的计算机系统和/或其他设施进行连接。

（4）故障模式描述：描述系统故障而非系统支持的设备故障造成的影响的文件。故障模式描述文件应涵盖以下方面：

① 需要评估的故障列表；及

(2) 对每一故障的系统响应；

③ 对每一故障后果的评价。

（5）参数化：通过更改参数来配置和调整系统和软件功能，通常不需要计算机编程。参数化一般由系统供应商或服务提供方而非操作人员或终端用户完成。

（6）可编程设备：安装有软件的物理元件。

（7）鲁棒性：对异常输入和条件作出响应的能力。

（8）模拟测试：监测、控制或安全系统测试，受控设备部分或完全由模拟工具取代，或者通信网络和线路部分由模拟工具取代。

（9）软件模块：一段独立的代码，提供指定且紧密耦合的功能。

（10）软件主文件：构成软件初始源的计算机文件。对于定制软件来说，可能是可读的源代码文件，而对于现有商用软件来说，可能是不同形式的二进制文件。

（11）软件结构：不同软件模块如何交互的概述，通常称为软件体系结构或软件层次结构。

(12)子系统：系统的可识别部分，完成一项特定功能或一整套特定功能。

（13）系统：元件、设备和逻辑的组合，具有明确的用途、功能和性能。在本节中，系统由系统供应商交付。

（14）集成系统：由多个系统组成的系统，通常来自不同的供应商。在本节中，集成系统作为船舶的一部分，包括由船厂交付的所有监测、控制和安全系统。

（15）业主：在船舶建造期间，业主为订购船舶的组织或个人，在船舶营运期间，业主

为拥有或管理船舶的组织。在本节中，业主是一个有明确职责的角色。

（16）供应商：通用名词，代指任何承包或分包提供服务、系统元件或软件的组织或个人。

（17）服务供应商：非 IACS 成员雇佣的，由设备制造商、船厂、船东或其他与检查工作相关的客户要求的并为船舶或海上移动平台提供诸如安全系统和设备的测量、试验或检修的，其结果影响到验船师对签发船舶入级或法定证书以及服务做出决定的个人或公司。

（18）系统供应商：在系统集成商的协调下承包或分包提供系统元件或软件的组织或个人。在本节中，系统供应商是一个有明确职责的角色。

（19）系统集成商：在计算机系统的整个生命周期，负责协调系统和子系统供应商之间互动的单个组织或个人，以便将这些系统集成到经验证的全船范围内的集成系统，同时为计算机系统提供适当的操作和维护服务。在本节中，系统集成商是一个有明确职责的角色。在设计和交付阶段，船厂是默认的系统集成商，在运营阶段，业主是默认的系统集成商。

（20）船舶：安装计算机系统的船舶或近海设施。

# 2.6.3 系统和元件批准

2.6.3.1 实现船舶功能所必需的Ⅱ类或Ⅲ类计算机系统应持有产品证书。产品证书的目的是确认该系统的设计和制造已经完成，并符合适用的入级要求。产品证书包括两项主要的验证：

（1）对计算机系统的文件进行评估，包括表2.6.4.1（1）所列文件；

（2）对将要上船的系统进行检验和测试。

在满足要求的条件下，CCS可接受替代验证方案，并颁发产品证书。

2.6.3.2 常规制造和包括标准化软件功能的计算机系统，可根据CCS相关要求进行型式认可。型式认可包括两项主要的验证：

（1）对型式认可文件进行评估，包括表2.6.4.1（1）所列文件以及申请CCS型式认可所需的其他文件；

（2）对标准化功能进行检验和测试。

当因为特定船舶的功能、参数配置和安装元件需要进行实船验证时，即使系统完成型式认可，仍需要持有产品证书。

# 2.6.7 系统供应商的要求

2.6.7.1 系统供应商应形成质量计划（表2.6.4.1（1）第1项）文件，阐明质量管理体系适用于将要交付的具体系统的设计、施工、交付和维护。表2.6.6.2中与系统供应商相关的项目都应被证明存在且被执行。

2.6.7.2 应采用能够唯一标识系统及其不同软件模块和同一软件模块的不同版本的方法，该方法应适用于系统和软件的整个生命周期，该方法的相关文件通常是质量计划的一部分。系统和软件唯一标识的技术要求见本节 2.6.11.1 条。

2.6.7.3 应在系统说明（表 2.6.4.1（1）第 2 项）中明确系统的规格和设计。除了作为详细设计和实施的说明外，系统说明的目的是确保整个系统交付可依据该说明，且符合适用的规范和条款。系统说明应包含以下信息：

（1）目的和主要功能，包括任何安全方面；

（2）定义的系统类别；

（3）关键性能特征；

（4）符合的技术要求和CCS规范；

（5）用户界面/镜像；

（6）通信和接口方面：与船舶其他系统的接口的识别和描述；

（7）硬件布置相关方面：

① 网络架构/拓扑，包括所有网络元件，如交换机、路由器、网关、防火墙等；

② 系统所有接口和硬件节点的内部结构（例如操作站、显示器、计算机、可编程设备、传感器、执行器、I/O模块等）；

③ I/O分配（将现场设备映射到信道、通信链路、硬件单元、逻辑功能）；

④ 电源布置；

⑤ 故障模式描述。

2.6.7.4 应按照CCS《电气电子产品型式认可试验指南》对系统和子系统的硬件进行环境测试，并向CCS提交测试结果（表2.6.4.1（1）第3项）。

2.6.7.5 应开发为交付项目创建、变更或配置的软件，并根据质量计划中所选定的标准对质量保证活动进行评估。质量保证活动可以在软件结构的几个级别上进行，并应适当地包括定制的软件和配置的组件（例如软件库）。对于Ⅱ类和Ⅲ类系统中的模块，所有已执行的审查、分析、测试和其他验证的范围、目的和结果应记录在测试报告（表2.6.4.1（1）第4项）中。

如采用黑盒测试方法①，则对软件的验证应至少包含以下几个方面：

(1) 所有软件模块的参数化和配置的正确性、完整性和一致性；

(2) 预期功能；

(3) 预期鲁棒性。

2.6.7.6 在工厂测试前尽可能对系统进行测试。系统测试的主要目的是让系统供应商验证整个系统的交付符合规范、批准的文件和适用的法规，以及系统已完成并准备好进行工厂

测试。应尽可能真实地模拟故障，以验证系统的故障检测和响应能力。一些测试可以通过使用模拟器和同型硬件来执行。测试环境应被记录下来，包括对任何模拟器、仿真器、测试存根、测试管理工具或其他影响测试环境的工具及其限制条件的描述。测试用例和测试结果应分别记录在测试程序和测试报告（表2.6.4.1（1）第5项）中。系统测试应至少验证系统的以下方面：

（1）功能；

（2）故障和故障的影响（包括诊断功能、检测、报警、响应）；

（3）性能：

（4）软件和硬件之间的集成；

（5）人机接口；

（6）与其他系统的接口。

2.6.7.7 系统应安排工厂验收测试。工厂验收测试的主要目的是向 CCS 证明该系统已经完成并符合适用的入级规则，从而获取船级证书。工厂验收测试程序（表 2.6.4.1（1）第 6 项）应选取本节 2.6.7.6 所述系统测试中具有代表性的测试项目，包括正常的系统功能测试和故障响应测试。对于 II 类和 III 类系统，还应进行网络测试，以验证本节 2.6.11.2 中的网络韧性要求。若征得各方同意，网络测试可以作为船上系统测试的一部分进行。工厂验收测试通常应使用项目专用软件在船上实际安装的硬件上执行，具备必要的用于功能模拟和故障响应的工具或手段，对于其他测试方案，如采用同型硬件或模拟硬件（仿真器）需经 CCS 同意。对于每个测试用例，应记录测试是否通过或不通过，并应将测试结果记录在测试报告（表 2.6.4.1（1）第 7 项）中。测试报告还应包含在执行测试时已安装在系统中的软件（包括软件版本）的列表<sup>①</sup>。

2.6.7.8 系统软件模块的初始安装和后续更新应按照系统供应商和系统集成商共同商定的变更管理程序进行。变更管理程序应符合本节2.6.10的相关要求。网络安全措施应符合CCS《船舶网络安全指南》相关要求。

# 2.6.8 系统集成商的要求

2.6.8.1 在本节中，船厂被视为船舶开发和交付阶段的系统集成商，除非明确指定了其他组织或个人。

2.6.8.2 系统集成商应形成质量计划（表2.6.4.2（1）第1项）文件，阐明质量管理体系适用于将要安装在船上的系统的安装、集成、完成和维护。表2.6.6.2中与系统集成商相关的项目都应被证明存在且被执行。

2.6.8.3 对于交付到具体船舶的系统，应根据本节 2.6.5.2 定义的系统故障的影响确定系统所属类别（表 2.6.4.1（2）第 2 项）。系统的类别信息需传达给相关的系统供应商。必要时，CCS 可要求通过风险评估（表 2.6.4.1（2）第 3 项）来确定系统类别。

2.6.8.4 应对集成系统予以规定并形成系统结构说明（表 2.6.4.1（2）第 4 项）。系统结构说明通过分配单个系统的功能和定义系统之间的主要接口，为系统的分类和不同集成系统的开发奠定了基础<sup>⑧</sup>，也为本节 2.6.8.6 要求的集成系统在船上的测试提供了依据。

船舶的系统结构说明至少应包含以下描述：

（1）概述整个系统结构（集成系统）；

（2）每个系统的目的和主要功能；

（3）不同系统之间的通信和接口。

2.6.8.5 应在船上进行系统验收测试。系统验收测试的主要目的是验证系统在安装后，以及与船上其他机械/电气/处理系统，包括与其他控制和监测系统之间可能存在的接口集成后，其功能运行情况。对于每个测试用例，应记录测试是否通过或不通过，并应将测试结果记录在测试报告（表 2.6.4.1（2）第 6 项）中。测试报告还应包含在执行测试时已安装在系统中的软件（包括软件版本）的列表。

2.6.8.6 集成测试应在不同系统在船上的最终环境中安装和集成后进行<sup>①</sup>。集成测试的目的是验证集成系统完整安装后的功能，包括所有接口和相互依赖关系符合要求和规定。集成测试至少应验证系统的以下方面：

（1）集成系统的整体功能；

（2）系统之间的故障响应；

（3）性能；

（4）人机接口；

（5）不同系统之间的接口。

2.6.8.7 系统集成商应遵循本节 2.6.10 系统变更管理程序（表 2.6.4.1（2）第 9 项）。

# 2.6.9 计算机系统的维护要求

2.6.9.1 在本节中，业主被视为船舶运行阶段的系统集成商，除非其明确指定了其他组织或个人。业主应及时告知CCS其指定的系统集成商，系统的任何变更应由系统集成商和系统供应商共同负责。

2.6.9.2 系统集成商应确保船上存放必要的软件和硬件变更管理程序，任何软件的修改/升级应按照该程序进行。变更管理的具体要求见本节 2.6.10。

2.6.9.3 系统集成商应记录运行阶段计算机系统的变化。记录内容应包含相关软件版本和本节 2.6.10.9 中的信息。

2.6.9.4 系统供应商应遵循系统维护程序，包括本节2.6.10中的变更管理程序。

2.6.9.5 系统供应商应确保在对船上的系统进行变更之前，对系统的计划变更已经通过了相关的内部测试。

# 2.6.10 变更管理

2.6.10.1 变更管理应由各利益相关方在计算机系统生命周期的不同阶段共同协商进行。如系统所需的变更已经获得各方的批准（在工厂测试阶段，通常指系统集成商和CCS），则应遵循定义的变更管理程序。

2.6.10.2 变更管理程序涵盖硬件和软件。工厂验收测试之后，由系统供应商按照程序负责系统的所有变更。例如验证购买的软件的新版本、新硬件、修改的控制逻辑、可配置参数的变更。变更管理程序应至少包含本节 2.6.10.3 至 2.6.10.10 的内容。

2.6.10.3 系统供应商应确保每个系统和软件版本都能够被唯一识别，见本节2.6.7.2。

2.6.10.4 应建立机制，用于对构成软件模块主文件的文件进行处理。应明确人员权限，同时通过工具和机制来确保主文件的完整性。

2.6.10.5 应明确规定如何对船载计算机系统的软件模块进行备份和恢复。

2.6.10.6 在对系统进行变更前应进行影响分析，变更前影响分析的结果将决定变更活动的执行程度。变更前影响分析应能：

（1）确定变更的后果；

（2）确定对现有文件的影响；

（3）确定所需的验证和测试活动；

（4）将变更情况告知其他利益相关方；

（5）在变更前获得其他利益相关方的许可（如CCS和/或业主）。

2.6.10.7 当维护过程包括在系统中安装新版本的软件时，应能将软件回滚到以前安装的版本，以使系统恢复到已知的稳定的状态。应对回滚操作进行记录和分析，以便找到并消除缺陷根源。

2.6.10.8 系统变更应尽可能在船上安装前进行验证。安装后，应根据制定的验证程序进行验证，包括：

（1）新功能和/或改进是否达到预期效果；

（2）回归测试，验证修改不会对预期不受影响的功能或能力产生任何负面影响。

2.6.10.9 系统和软件的变更应记录在变更记录中，以确保变更的可见性和可追溯性。变更记录应至少包含以下项目：

（1）变更的目的；

（2）变更和修改的描述；

（3）变更前影响分析的主要结论（本节2.6.10.6）；

（4）所有新系统或软件版本的标识和版本（本节2.6.10.3）；

（5）测试报告或测试总结（本节2.6.10.8）。

2.6.10.10 软件变更可以记录在计划维护系统、软件注册表或同类文件中。

2.6.10.11 船舶营运阶段，CCS对变更管理的验证一般在船舶年度检验时进行。检验时，应提供变更管理程序和相关变更记录。如果变更需要提前得到CCS批准，则变更相关的程序和文件可在该阶段进行验证。船舶建造阶段，CCS对变更管理的验证分为两部分：

（1）变更管理程序作为表2.6.6.2中质量管理体系验证的一部分进行验证；

（2）在工厂验收测试期间和之后，结合变更管理程序在具体项目中的实施情况进行验证。

# 2.6.11 计算机系统的技术要求

2.6.11.1 系统应提供识别其名称、版本、标识和制造商的方法。建议系统按照国际标准 ISO 24060 的规定，自动向船舶软件日志记录系统（SSLS）报告其软件的状态。

2.6.11.2 风险评估分析/FMEA中应明确数据链路失效的状况，包括：

（1）III类系统数据链路的单一故障不应导致船舶功能的丧失。船舶功能故障的任何后果应符合故障安全原则；

（2）对于Ⅱ类和Ⅲ类系统，远程控制系统中的任何功能损失都应通过本地/手动方式进行补偿；

（3）数据链路应能防止或应对通信速率过高；

（4）数据链路应具有自检功能，检测自身链路故障和链路连接的节点的通信故障；

（5）检测到故障时应发出报警。

2.6.11.3 III类系统不应采用无线数据链路，除非经CCS依据其可接受的国际或国家标准进行工程分析后特别考虑。

2.6.11.4 其他类别的系统可采用无线数据链路，但应满足以下要求：

（1）应使用认可的国际无线通信系统协议，并满足以下要求：

①信息完整性。故障预防、探测、诊断和修正，以便收到的信息（与发送的信息相比较）不被破坏或更改；

(2)配置和设备验证。应仅允许与系统设计中包含的设备连接；

③信息加密。保护机密和/或关键数据内容；

④ 安保管理。保护网络资产，防止未经授权访问网络资产。

(2) 船舶内部无线系统应满足国际电信联盟和船旗国主管机关对无线频率和功率水平的要求；

（3）系统操作应考虑到港口和当地法规在射频传输方面的规定，因频率和功率的限制而禁止使用无线数据通信链路；

（4）无线数据通信设备应在系泊试验和航行试验期间进行测试，证明在预期的操作条件下，射频传输不会因电磁干扰引起自身和其他任何设备的故障。

2.6.11.5 本节2.6.11中技术要求的实施应作为系统说明、工厂验收测试、系统验收测试的一部分进行验证。

# 第7节 传感器

# 2.7.1 一般要求

2.7.1.1 传感器应能长期稳定地正常工作。其量程及频率特性（如适用时）应与被测参

数的预计的最大变化范围及变化速率相适应，并应具有适当的精度和灵敏度。

2.7.1.2 传感器应在其安装位置对环境条件有良好的适应性。传感器应坚固耐用，具有良好的机械保护、可靠的电气连接和良好的绝缘性能。

2.7.1.3 传感器的安装位置应能正确反映出被监测参数，并应易于接近、测试和拆装。为便于维修和更换，传感器应加装防护套。在难以接近和拆装的位置，还应另装1个备用传感器。

2.7.1.4 a类保护动作应设置独立于b类和c类保护动作的传感器；如无特殊的要求，b类保护动作可与显示、报警合用1只传感器，c类保护动作可与其本身的报警合用1只传感器。

# 第8节 驾驶室遥控主推进装置

# 2.8.1 一般要求

2.8.1.1 如主推进装置由驾驶室遥控，则应满足本章2.8.1.2至2.8.1.12的要求。

2.8.1.2 当船舶在海上航行（包括机动操纵）时，主推进装置的推进方向、推进速度及螺旋桨螺距（如设有时）应可从驾驶室完全控制。

2.8.1.3 在驾驶室对主推进装置的遥控，每一独立螺旋桨应使用单一控制装置进行控制，所有相关装置应自动进行操作，必要时，包括防止推进机械超负荷和防止在振动转速禁区内长期运转的装置。如果多螺旋桨设计为同时运行，则可由一个控制装置进行控制。

2.8.1.4 驾驶室的控制系统应独立于其他用途的发信系统，但亦可用一根控制杆用于控制两个系统。

2.8.1.5 从驾驶室控制装置发出的各种操作指令，包括在应急情况下以最大前进营运航速变为全速后退，其自动顺序控制应在主机所允许的时间间隔内完成操作。

2.8.1.6 驾驶室控制站应设置独立于驾驶室控制系统的主推进装置紧急停车装置，但其执行机构可不要求独立，且其布置应能防止被误触动。

2.8.1.7 驾驶室发出的主推进装置指令应在机舱集控站（如设有）和主推进装置就地控制站显示。

2.8.1.8 当推进装置在驾驶室遥控起动，而由于推进装置所处的状况可能产生危险时，如盘车机构正啮合着、滑油压力已下降等，则遥控起动应能自动被阻止。

2.8.1.9 驾驶室控制系统应设计成使其能在发生故障时发出报警信号，在这种情况下，除非认为实际上不可行，否则，螺旋桨的转速和转向应一直保持到进行就地控制为止，特别当监控系统的动力源（电力、气动、液压）中断或控制转换时，应不会导致推进功率或推进转向发生较大和突然的变化。

2.8.1.10 应在驾驶室和机器处所设有报警装置，对主推进装置的最低起动空气压力达到临界值时发出报警。如果主推进装置遥控系统具有自动起动功能，连续自动起动的次数应不超过3次，当3次起动失败后，应在驾驶室发出视觉和听觉报警信号。

# 2.8.1.11 驾驶室控制站应显示：

（1）主机转速或螺旋桨转速；

（2）定距桨的转向或调距桨的桨角（或螺距）；

（3）离合器、轴制动器的状态（如适用时）；

（4）正在实施控制的控制站；

（5）驾驶室控制站有关的监控系统动力源。

2.8.1.12 如主推进装置由驾驶室遥控，自动控制系统应设计为：主推进装置因即将发生的减速或停车而引发的临界报警能确保及时准确地提供给负责航行值班的驾驶员，以帮助其评估紧急情况下的航行条件。尤其是该自动控制系统在进行控制、监视、报告、报警以及采取减速或停车的安全措施的同时，还应能为负责航行的值班驾驶员提供实施手动的干预（如越控），但因手动干预而导致机器和/或主推进装置短时间内完全失效（如超速）的情况除外。

2.8.1.13 驾驶室控制站对主汽轮机的控制应满足本篇3.2.3.2（1）的要求，但当机器处所有人值班时，低速盘车可设置为手动操作。

# 第3章 周期无人值班机器处所AUT-0附加标志的要求

# 第1节 一般规定

# 3.1.1 适用范围

3.1.1.1 本章适用于主推进装置由驾驶室控制站遥控、机器处所包括机舱集控站（室）周期无人值班的船舶，其自动化附加标志为AUT-0。

3.1.1.2 具有AUT-0自动化附加标志的船舶除应满足本篇第1章和第2章的要求外，还应满足本章的要求。

3.1.1.3 具有AUT-0自动化附加标志的客船，还应注意到船旗国主管机关的要求（如有时）。

3.1.1.4 在无人值班周期内，自动化系统应能保证下列机电设备连续正常运行：

（1）主推进装置，包括主机（如主柴油机、主汽轮机、主燃气轮机和电力推进装置等）、传动装置（如离合器、减速齿轮箱等）以及推进器（如调距桨等）；

（2）为主推进装置服务的重要辅机；

（3）主、辅锅炉；

（4）电站；

（5）其他机电设备，如空气压缩机、舱底水系统（包括油水分离器）、与遥控有关的阀、燃油系统以及CCS认为有必要进行自动化监控的其他机电设备。

# 第2节 主推进装置

# 3.2.1 一般要求

3.2.1.1 应能在机舱集控站（室）对主推进装置的起动、调速、正倒车换向（包括轴系换向装置）、停车进行遥控和自动顺序控制。当控制系统出故障时，应能保证在就地控制站对主推进装置由人工进行上述操纵。

# 3.2.2 主柴油机

3.2.2.1 主柴油机应满足本规范第3篇第9章的有关要求。

3.2.2.2 主柴油机控制系统应满足下列要求：

（1）制系统应保证主柴油机及轴系在所有操纵状态下不会受到有害的机械负荷和热负荷；

（2）主柴油机的自动顺序控制应设有必要的联锁，以防止机械损伤；

(3) 主柴油机连续自动起动的次数应不超过 3 次, 当第 3 次起动失败后, 应停止自动再起动, 并发出视觉和听觉报警信号, 同时该单项报警信号还应延伸至驾驶室控制站;

（4）控制系统应能保证主柴油机自动快速地越过振动转速禁区；

（5）当主柴油机用于主推进时，应设有使起动空气压力保持在所要求的压力范围内的措施。

# 3.2.2.3 主柴油机安全系统应满足下列要求：

（1）安全保护：

①当所要求的备用泵自动起动时应发出报警；

(2)如对要求自动降低功率的安全保护设有越控功能时, 应有措施防止越控功能被误触动, 实施越控时应发出报警;

(3)如对要求自动停车的安全保护设有越控功能时（超速时除外），应有措施防止越控功能被误触动，实施越控时应发出报警。主柴油机自动停车时，只有人工复位之后，例如使遥控手柄操至“停车”位置，才可以使恢复正常工作状态的主柴油机重新起动。否则，柴油机不应进行自动再起动；

（2）主柴油机安全系统的要求见本章表3.10.1.1第4栏；

（3）在驾驶室和机舱集控站（室）还应设置紧急停车按钮（另见本篇第2章2.8.1.6和本章3.7.3.5）。

3.2.2.4 主柴油机报警系统（包括显示）应满足下列要求：

（1）本章表3.10.1.1所要求的遥测显示仅适用于在机舱集控站（室）显示和报警的项目，见表3.10.1.1第2栏和第3栏；

（2）应在驾驶室控制站报警的项目和报警方式见本章表3.10.1.1第5栏，显示项目见本篇第2章2.8.1.11。

# 3.2.2.5 如主柴油机采用减振器：

对于采用主机润滑油循环供油并靠其阻尼工作的扭振减振器或纵振减振器，其自动化监视项目要求应满足本章表3.10.1.1的有关要求。

# 3.2.3 主汽轮机

3.2.3.1 主汽轮机应满足本规范第3篇第7章的有关要求。

3.2.3.2 主汽轮机控制系统应满足下列要求：

（1）主汽轮机应能低速自动盘车，以防止汽轮机在机动操纵时由于转子停转超过要求时间而产生热变形，但在驾驶室应有可能中止此种低速自动盘车的装置；

（2）控制系统应能保证主汽轮机自动快速地越过振动转速禁区，也可采取替代措施，如当转速进入转速禁区内时，应发出报警，以使人员尽快操纵主汽轮机快速越过转速禁区。

3.2.3.3 主汽轮机安全系统的要求见本章表3.10.1.1第4栏。

3.2.3.4 主汽轮机报警系统（包括显示）应满足下列要求：

（1）应在机舱集控站（室）显示和报警的项目见表3.10.1.1第2栏和第3栏；

（2）应在驾驶室控制站报警的项目和报警方式见本章表3.10.1.1第5栏，显示项目见本篇第2章2.8.1.11。

# 3.2.4 主燃气轮机

3.2.4.1 主燃气轮机应满足本规范第3篇第8章的有关要求。

3.2.4.2 主燃气轮机控制系统应满足下列要求：

（1）除应急操纵情况外，燃气轮机从起动开始功率增大到额定值，以及从额定功率减小到停车，其燃油的油量控制应使涡轮机的任何部件都不致发生危险的热应力和/或机械应力，正常运行时，燃气轮机的遥控系统应使燃气轮机的排气温度和转子转速保持在设定范围内，且在整个控制范围内燃烧室的燃烧应保持稳定；

(2) 燃气轮机遥控系统应设计成使涡轮机滑油系统、燃油系统和冷却系统等具有适当的联锁, 以使燃气轮机按预定的起动和停车程序工作, 顺序控制应满足下列要求:

①燃气轮机在起动前和停车后，滑油泵应进行工作，但对于装有滚柱轴承和设有机带滑油泵的燃气轮机可予以特殊考虑；

(2) 点火前, 燃气轮机的燃烧室应以空气进行充分扫气;

③主燃油阀的开启不应先于点火火花之前出现；

(4) 每一燃烧器的点火过程应不超过要求的时间, 而且, 如燃气轮机在要求时间内不能成功起动, 则应停止进行再起动;

⑤点火时燃烧室内不应喷入过量的燃油。

（3）多台燃气轮机用作单桨的推进装置，由遥控系统进行控制时，应使每台燃气轮机的负荷分配不会形成异常的不平衡。

3.2.4.3 主燃气轮机安全系统的要求见本章表3.10.1.1第4栏。

3.2.4.4 主燃气轮机报警系统（包括显示）应满足下列要求：

（1）应在机舱集控站（室）显示和报警的项目见本章表3.10.1.1第2栏和第3栏；

（2）应在驾驶室控制站报警的项目和报警方式见本章表3.10.1.1第5栏，显示项目见本篇第2章2.8.1.11。

# 3.2.5 调距桨

3.2.5.1 调距奖应满足本规范第3篇第11章第3节的有关要求。

3.2.5.2 在驾驶室控制站应能对调距桨的桨角（或螺距）进行遥控和自动顺序控制。

3.2.5.3 对每个独立的调距桨以及与之相连的自动操作设备，均应由1个单一的控制器进行控制。可同时操作的多桨系统，可以由1个控制设备控制。

3.2.5.4 主推进调距桨与主机的联合控制系统，应能防止主机和轴系过载，以及因过载或超速引起的停车。

3.2.5.5 主推进调距桨的遥控液压系统应设有备用动力源。当液压系统压力低于预定工作压力时，备用液压泵应能自动起动。

3.2.5.6 调距桨的桨角（或螺距）和桨轴转速应在对其控制的各控制站予以显示（见本篇第2章2.8.1.11和表3.10.1.1）。

3.2.5.7 对调距奖的报警项目见本章表3.10.1.1。

# 3.2.6 电力推进装置

3.2.6.1 电力推进装置的监控功能应满足本规范第8篇第15章第2节的有关要求。

3.2.6.2 驾驶室控制站应能操纵控制电力推进装置，并应设有指示推进轴转向、转速及其他必要运行参数的仪表。

# 3.2.7 离合器

3.2.7.1 离合器的自动化监视项目要求应满足本章表3.10.1.1的有关要求。

# 3.2.8 齿轮箱

3.2.8.1 齿轮箱的自动化监视项目要求应满足本章表3.10.1.1的有关要求。

# 第3节 锅 炉

# 3.3.1 主锅炉

3.3.1.1 主锅炉应满足本规范第3篇第6章的有关要求。

3.3.1.2 主锅炉的控制系统应满足下列要求：

（1）水位控制

①自动控制系统应保证锅炉水位在所有工况下自动保持在要求的范围内；

(2)每台锅炉应设有 2 个独立的低水位传感器（其中 1 个用于检测过低水位的安全系统）和 1 个高水位传感器，应防止因摇晃使锅炉水位传感器产生误动作信号。

（2）燃烧控制

(1) 燃烧过程的自动控制系统应能在主汽轮机所有工况下保持预定的蒸汽量、蒸汽压力、蒸汽温度以及稳定的燃烧;

②顺序控制的自动点火锅炉应满足下列要求：

(a) 锅炉的炉膛和烟道在喷油器开始点火前应进行定时的前扫气, 扫气时间应足以保证炉膛和烟道 4 次换气, 扫气时应使空气调节门处于全开位置;

(b) 点火应在空气进入燃烧室并完成前扫气后方可进行, 喷油器进油阀应在点火火花出现之后方可打开, 如点火不着, 点火装置和喷油器进油阀应自动关闭, 进油阀从开启到关闭的时间不得大于 15s;

(c) 应设有火焰监测器，当故障熄火时能自动关闭喷油器的进油阀，关闭时间应不迟于熄火后 6s。

# 3.3.1.3 主锅炉安全系统：

（1）当发生下列故障之一时，应自动切断锅炉的燃油供给或采用其他等效措施紧急停炉：

(1)炉膛熄火（指所有喷油器的火焰都熄灭）或点火失败；

②锅炉水位低于极限低水位（极限低水位不得低于锅炉的最低安全水位或水位表能见的水位）；

③供锅炉炉膛燃烧的空气失压。

（2）除（1）要求外，在机舱集控站（室）还应设置手动紧急切断每台锅炉燃油供给的设施（另见本章3.7.3.5）；

（3）为锅炉服务的燃油泵，当压力低于预定工作压力时，应自动起动备用泵并投入工作；

(4) 为锅炉服务的给水泵, 当压力或流量低于设定值时, 应自动起动备用泵并投入工

作。

# 3.3.1.4 主锅炉报警系统（包括显示）：

（1）应在机舱集控站（室）显示和报警的项目见本章表3.10.1.1第2栏和第3栏；

（2）应在驾驶室控制站报警的项目及报警方式见本章表3.10.1.1第5栏。

# 3.3.2 辅锅炉

3.3.2.1 辅锅炉应满足本规范第3篇第6章的有关要求。

3.3.2.2 辅锅炉的燃烧控制一般应满足本节3.3.1.2（2）的要求。

3.3.2.3 辅锅炉的安全系统应满足本节 3.3.1.3（1）、3.3.1.3（2）和 3.3.1.3（3）的要求。为辅锅炉服务的给水泵，当锅炉水位低于设定值时，应自动起动备用泵并投入工作。

3.3.2.4 对辅锅炉的自动化监视项目应满足本章表3.10.1.1的有关要求。

# 3.3.3 废气锅炉

3.3.3.1 废气锅炉应满足本规范第3篇第6章的有关要求。

3.3.3.2 废气锅炉的自动化监视项目应满足本章表3.10.1.1的有关要求。

3.3.3.3 机舱集控室应设有锅炉水泵运行指示灯。

3.3.3.4 废气锅炉的安全系统应保证当汽鼓蒸汽出口压力/温度过高时，应触发废气旁通或蒸汽释放机构动作。

# 3.3.4 热油加热器

3.3.4.1 热油加热器应满足本规范第3篇第4章第8节的有关要求。

3.3.4.2 热油加热器的自动化监视项目应满足本章表3.10.1.1的有关要求。

3.3.4.3 当用于驳运燃油的循环泵排出压力低于预定值时，应能自动起动备用泵。

3.3.4.4 为保证加热器在整个运行过程中维持稳定的工况，下列加热器设备应设置自动控制：

# （1）燃烧系统；

（2）燃油温度或粘度（仅适用于重油）；

（3）热油温度。

# 第4节 电 站

# 3.4.1 一般要求

3.4.1.1 电站除应满足本节要求外，还应满足本规范第4篇第2章的有关要求。

3.4.1.2 电站的自动化监视项目应满足本章表3.10.1.1的有关要求。

# 3.4.2 供电连续性

3.4.2.1 在通常由一台发电机组供电的情况下，应采取如卸载（见本节3.4.2.6）或自动起动备用机组等措施，以保证船舶在包括机动航行在内的所有航行工况下，具有与有人值班机器处所同等的安全性。

3.4.2.2 在通常需由多台发电机组并联运行供电的情况下，则应采取如卸载（见本节3.4.2.6）或将汇流排分段等措施，以保证在这些发电机组中的一台停止工作时，其余发电机组仍能保持对推进、操纵和船舶安全所需设备的供电而不过载。

3.4.2.3 在失电情况下，备用发电机组应能自动起动，并自动连接至主配电板。备用发电机组的容量，应足以对包括机动航行在内的所有航行工况下所必需的设备供电，以保证船舶具有与有人值班机器处所同等的安全性。

3.4.2.4 失电后恢复供电时，电动机驱动的为主推进装置服务的重要辅助机械，应能自动按一定的程序再起动。

3.4.2.5 备用发电机组及其自动起动系统，应具有在  $45\mathrm{s}$  的时间内承载额定负载的性能。

3.4.2.6 为了对重要设备安全供电，应设有适当的卸载装置，以能在下列情况下将非重要负载自动切断：

（1）当由于自动接入附加负载而引起总负载超过发电机额定输出功率时；

（2）当多台发电机组并联运行供电，其中一台发电机发生故障，而使总负载超过仍在供电的其余发电机组额定功率之和时。

# 第6节 其他设备

# 3.6.1 尾管轴承

3.6.1.1 尾管轴承的自动化监视项目要求应满足本章表3.10.1.1的有关要求。

# 3.6.2 阀

3.6.2.1 本节所涉及的阀系指用于舱底水系统和海水系统（包括压载水系统）的阀。

3.6.2.2 阀执行机构的动力源故障不应使阀和船舶处于不安全的状态。

3.6.2.3 对阀进行控制的控制站（室）应指示出阀的开度或开、闭状态。

3.6.2.4 位于可能被水浸没位置的阀的操作机构，应在被浸没时也能进行正常操作。

3.6.2.5 当遥控或自动控制出现故障时，应能对阀进行手动操作。

# 3.6.3 净油机等其他设备

3.6.3.1 净油机、污油柜、焚烧炉等的自动化监视项目要求应满足本章表3.10.1.1的有关要求。

# 3.6.4 油柜

3.6.4.1 日用燃油柜和沉淀柜的容量应与无人值班的周期（如8h，16h，24h等等）相适应，并应至少增加  $10\%$  的容量。日用燃油柜应设置低油位报警（见本章表3.10.1.1），而沉淀柜应设置高、低油位报警。

3.6.4.2 如日用燃油柜采用遥控或自动加油方式，则应有防止溢油的措施，并设置高、低油位报警延伸至驾驶室控制站；而上述3.6.4.1的容量要求则可以免除。

# 第7节 控制站（室）的附加要求

# 3.7.1 控制站（室）的设置

3.7.1.1 机器处所周期无人值班的船舶一般应设置驾驶室控制站、机舱集控站（室）和就地控制站。

# 3.7.2 驾驶室控制站的功能

3.7.2.1 当船舶在海上航行（包括机动操纵）时，驾驶室控制站应能对主推进装置的推进方向和推进速度及其他有关装置（如设有时）实施有效的控制。必要时，该控制应能在任

任何时候被转换到其他控制站（室），且控制转换应满足本篇2.5.2的有关要求。驾驶室控制站的设计和布置应满足本篇第2章第8节的要求。

# 3.7.2.2 驾驶室控制站的报警：

（1）驾驶室控制站报警的项目和报警方式见本章表3.10.1.1第5栏，被监控的机电设备和自动化监控系统的故障报警一般可以4种方式延伸到驾驶室控制站：安全系统动作的组合报警、单独报警、严重故障的组合报警和一般故障的组合报警；

（2）在驾驶室控制站的听觉报警信号允许在报警被应答后消声，但视觉报警信号应在故障排除后消除，而且，视觉报警信号应能在故障报警被应答前后有所区别。

3.7.2.3 如主机由驾驶室控制站遥控起动时，驾驶室控制站应显示主机起动空气压力或起动蓄电池组电压。

# 3.7.3 机舱集控站（室）的功能

3.7.3.1 机舱集控站（室）应能实施本节3.7.2.1要求的对主推进装置的控制功能和控制转换。此外，还应能对本章3.1.1.4所述的所有其余的机电设备进行自动化监控。

3.7.3.2 机舱集控站（室）应设置仪表或显示器，以指示确保机电设备安全可靠运行所必需的重要参数。要求显示的项目见本章表3.10.1.1第2栏。

3.7.3.3 机舱集控站（室）应对机电设备和自动化监控设备发生的所有故障，包括同时发生的数个故障均能报警。报警的具体项目见本章表3.10.1.1第3栏。

3.7.3.4 机舱集控站（室）内的听觉报警信号应在该站（室）内应答后方能消除。

3.7.3.5 机舱集控站（室）应设置独立于自动化系统的主机紧急停车按钮、锅炉及焚烧炉的紧急停油按钮，但其执行机构可不要求独立，其布置应能防止被误触动。

3.7.3.6 如对主机设有本篇1.1.2.1（9）所述的越控功能时，机舱集控站（室）亦应能实现该功能，并应有措施防止越控功能被误触动。

# 3.7.4 就地控制站的功能

3.7.4.1 就地控制站应满足本篇2.5.1.5的要求。

# 3.7.5 通信

3.7.5.1 除需满足本规范第3篇和第4篇的有关要求外，机舱集控站（室）或就地控制站（如适用时）及驾驶室控制站与轮机员起居处所之间应设有可靠的语音通信装置。

# 第8节 监控系统的附加要求

# 3.8.1 安全系统

3.8.1.1 所有被监控机电设备的安全系统应满足本篇第2章第3节的要求。

# 3.8.2 报警系统

3.8.2.1 报警显示的布置应有助于识别机器处所内故障发生的具体情况和位置。

3.8.2.2 如驾驶室的值班驾驶员是船上唯一的值班人员，在就地控制站监测到机器发生故障的情况下，报警系统应使该值班驾驶员知道下列情况：

（1）某一故障已经发生；

（2）发生的故障已被注意到（如应答、消声等）；

（3）故障已被消除。驾驶室控制站、轮机员起居处所和机器处所（机舱集控站或就地控制站）之间的其他通信措施可用来实现该功能。

3.8.2.3 可以采用组合报警的方式在驾驶室显示出所发生的故障。但对于推进装置的减速或自动停车的故障报警应能区分（见本章表3.10.1.1第5栏）。

3.8.2.4 应能在被监控机电设备正常运行时对报警系统进行检测。如切实可行，应在便于到达的位置设有对传感器进行试验的措施，而不妨碍机电设备的运行。

3.8.2.5 如报警已经应答，而且在第一次故障消除前又发生了第二次故障，此时应再次发出视觉和听觉报警信号。瞬时的故障发出的报警在应答前要予以保持。

3.8.2.6 机舱集控站（室）应设置重要参数和故障的自动记录设备。如使用计算机系统，

则可用计算机的记录设备代替。

3.8.2.7 应设置转换开关，使所有的故障报警均以组合报警方式延伸到轮机员公共处所和每个轮机员居住位置，以保证该报警信号能延伸至1个值班轮机员的舱室。

在机器处所（包括集控站（室））外对报警的应答不应消除在机器处所（包括集控站（室））内的视觉和听觉报警信号。

如1个警报在一定时间（最长不超过5分钟）内于机器处所（包括集控站（室））内未被应答，则应自动触发本规范第4篇第2章2.8.4所述的轮机员报警装置，并能在轮机员居住舱室和有关公共处所内清晰地听到。

# 第9节 消防与防止浸水

# 3.9.1 消防

3.9.1.1 周期无人值班机器处所（包括集控站（室））的消防措施应满足本规范第6篇的有关要求，特别应满足该篇对机器处所特殊布置、探火和失火报警、水灭火系统的要求。应通过设置一台能由驾驶室和消防控制站（如设有）遥控起动的主消防泵，或通过由主消防泵之一对消防总管系统给予永久性增压的方法，从消防总管系统立即供给适当压力的水。但对1600总吨以下的货船，如布置在机器处所的消防泵起动装置的位置易于到达，则不必满足上述要求。

3.9.1.2 周期无人值班机器处所的防火措施除了应满足本规范第3篇对一般机舱的要求外，还应满足下列要求：

(1) 燃油、滑油和液压油等易燃油管路的结构和布置，应使其能避免因振动和膨胀引起的过度应力，上述管路法兰接头应尽量减少；

（2）自动处理易燃油的设备，如油净化器、油加热器、滤油器、油泵等，均应有防止溢油的措施，如在下部设集油盘将漏油集中到集油柜，并应尽实际可能将上述设备集中布置在1个专门处所；

（3）对高压燃油管系应采取屏蔽或其他适当措施，以防止滴漏或喷溅出的油接触到热表面或进入机器的吸气口，如柴油机的高压燃油管应装设金属护套；

（4）应尽实际可能将高压燃油管的漏油收集到处于安全位置的集油柜，并对高压燃油管发生漏油予以报警，见本章表3.10.1.1；

（5）燃油或滑油加热器应满足下列要求：

①应设高温报警或低流量报警，除非不可能到达介质着火的温度；

(2)如加热介质的最高温度不能保证低于  $220^{\circ} \mathrm{C}$ , 则应设置独立的安全系统, 当油温过高或油流中断时, 能自动切断热源;

(3)如采用电加热器, 则应保证所有通电的电热元件浸没在油中, 并且电热元件表面温度应不超过  $220^{\circ} \mathrm{C}$ , 否则应由独立的安全系统自动切断电热器的电源。

# 3.9.1.3 对探火系统的特别要求

（1）在机器处所内应装设自动探火系统；

(2) 该系统应设计成具有自检的特性。在电源或系统发生故障时, 应能发出不同于火警的听觉报警;

(3) 探火指示屏应设于驾驶室、防火控制站、或其他易于到达而不会因机器处所失火导致引起指示器失效的处所;

（4）探火指示屏应能通过视觉信号，根据所布置的防火区指示出探测到的火警位置。听觉报警应遍及驾驶室及负责机器处所操作人员的起居处所，该听觉报警信号应明显区别于其他性质的听觉报警信号；

（5）探火器的类型和位置，应使其能迅速探测到机器处所在正常状况下的初始火灾。应注意防止误报警。该系统应能够对一种类型以上的火灾征兆作出反应，探测器的类型和处所位置应满足SOLAS公约和国际消防安全系统规则（以下简称“FSS规则”）以及CCS规范的有关要求，可采用组合形式的探测器；

（6）探测器区域的布置应使得操作人员能确定火源的地点。探火系统的布置、回路数量以及探头位置应根据每一种情况进行认可，并满足SOLAS公约和FSS规则的有关要求。机器所产生的气流应不会导致探测系统的失效；

（7）当探测器具有校正灵敏度的措施时，应设有必要的设施确保其调定值的固定和识别；

（8）如欲暂时关闭一路特殊回路或探测器时，该状态应有明确的标示。在给定间隔时间后，该回路或探测器的功能应能自动恢复；

（9）探火指示屏应设有能进行功能试验的装置；

（10）如果主电源失效，探火系统应通过独立的馈电线自动转至应急电源供电，并满足第4篇第2.9.6的要求；

（11）探火系统应在通往机炉舱入口的走廊、驾驶室及机舱控制站等位置提供手动触发火警的设备；

（12）该系统装船以后应进行试验，以确认系统设备的性能满足规范的要求。

# 3.9.2 防止浸水

3.9.2.1 机舱应设置舱底水高水位报警。报警水位应保证舱底水不致漫上双层底舱的顶板。

3.9.2.2 舱底水位传感器的数量及布置应能防止当船舶正常倾斜和摇摆时产生误报警。

3.9.2.3 海水进口、水线下排水或舱底喷射系统中能控制破损浸水的阀门的操纵位置，应充分考虑当水进入该处所时，人员到达并关闭阀门的时间至少需要  $10\mathrm{min}$  ；该操纵位置应布置在船舶满载时可能因管路破损导致该处所的浸水水位之上。

3.9.2.4 舱底污水阱的容量应足以容纳无人值班周期的正常泄水量。如舱底泵能自动起动，则可设置一个较小的舱底污水阱以容纳适当长时间的正常泄水量，但应对舱底进水量大于泵的排放量、泵运行时间过长或过于频繁起动的情况予以报警，其排放也应满足防污染的有关要求。有关监视项目见本章表3.10.1.1。

3.9.2.5 在机舱集控站、驾驶室控制站和轮机员起居处所应设有上述3.9.2.1和3.9.2.4所要求的报警。

# 第4章 有人值班机器处所自动化附加标志的要求

# 第1节 一般规定

# 4.1.1 适用范围

4.1.1.1 本章适用于机器处所（包括机舱集控站（室））有人值班的MCC和/或BRC自动化附加标志的船舶。

4.1.1.2 上述自动化标志的船舶除应满足本篇第1章和第2章的要求外，还应满足本章的相应要求。

# 第2节 MCC附加标志的要求

# 4.2.1 控制站（室）的设置

4.2.1.1 具有MCC附加标志的船舶应设置机舱集控站（室）和就地控制站。机电设备正常运行时，机舱集控站（室）连续有人值班。

# 4.2.2 机舱集控站（室）的功能

4.2.2.1 机舱集控站（室）的监控功能应满足下列要求：

（1）在机舱集控站（室）应能对主机的起动、调速、正倒车换向（包括轴系换向装置）、停车以及对调距奖的奖角（或螺距）进行遥控和自动顺序控制。必要时，该控制应能在任何时候被转换到其他控制站（室），且控制转换应满足本篇2.5.2的有关要求。此外，还应能对本篇3.1.1.4所述的所有其余的机电设备进行自动化监控。

（2）机舱集控站（室）的设计和布置应满足下列要求：

①在机舱集控站（室）对主推进装置的遥控，原则上应是以一单一的控制装置，对每一独立推进器完成控制，并自动完成配合服务装置的控制，必要时，包括防止推进装置超负荷和防止在振动转速禁区长期运转的装置；

(2)机舱集控站（室）的控制系统应独立于其他用途的发信系统，但亦可用一根控制杆控制两个系统；

(3)从机舱集控站（室）控制装置发出的各种操作指令，包括在应急情况下以最大前进营运航速变为全速后退，其自动顺序控制应在主机所允许的时间间隔内完成操作；

(4)当推进装置在机舱集控站（室）内遥控起动，而由于推进机械所处的状况可能产生危险时，如盘车机构正啮合着、滑油压力已下降等，则遥控起动应能自动被阻止；

⑤机舱集控站（室）控制系统应设计成使其能在发生故障时发出报警信号，在这种情况下，除非认为实际上不可行，否则，螺旋桨的转速和转向应一直保持到进行就地控制为止，特别当监控系统的动力源（电力、气动、液压）中断或控制转换时，应不会导致推进功率或推进转向发生较大和突然的变化。

（3）在机舱集控站（室）应设置仪表或显示器，以指示确保机电设备安全可靠运行所必需的重要参数。要求显示的项目见本章表4.2.6.1第2栏。

（4）在机舱集控站（室）应对被监控的机电设备和自动化监控设备发生的所有故障，包括同时发生的数个故障均能报警。报警的具体项目见本章表4.2.6.1第3栏。

（5）机舱集控站（室）内的听觉报警信号应在该站（室）内应答后方能消除。

（6）在机舱集控站（室）应设置独立于控制系统的主机紧急停车按钮和锅炉的紧急停油按钮，但其执行机构可不要求独立。此外，其布置应能防止被误触动。

（7）如对主机设有越控功能时，机舱集控站（室）亦应能实现该功能，并应有措施防止越控功能被误触动。

4.2.2.2 对本篇3.1.1.4所述的所有机电设备的自动控制和遥控功能，一般应满足本篇第3章第2节、第3节、第5节及第6节的要求。但允许人员在机舱集控站（室）内介入与人员能力相适应的某些控制过程，如以遥控代替部分项目的自动控制等。

4.2.2.3 安全系统中 a 类保护动作应能自动进行；b、c 两类保护动作可以自动也可以在

机舱集控站（室）内用人工操作来替代。

# 4.2.3 就地控制站的功能

4.2.3.1 就地控制站应满足本篇2.5.1.5的要求。

# 4.2.4 消防

4.2.4.1 消防措施一般应满足本篇3.9.1.3对探火和失火报警系统的要求（从集控站（室）内能看到机舱范围的探测器可以免设）以及3.9.1.2除（4）外的全部要求。

# 4.2.5 防止浸水

4.2.5.1 应设置舱底水高水位报警。

4.2.5.2 排放舱底污水应满足防污染的有关要求。

# 4.2.6 自动化监视项目表

4.2.6.1 具有MCC附加标志的自动化船舶，其自动化监视项目（如有时）应满足表4.2.6.1的要求。

# 第3节 BRC附加标志的要求

# 4.3.1 一般要求

4.3.1.1 具有BRC附加标志的船舶，应设置驾驶室控制站对主推进装置及其他有关装置等进行遥控，机器处所应连续有人直接看管本篇3.1.1.4所述的机电设备。

# 4.3.2 驾驶室控制站的功能

4.3.2.1 当船舶在海上航行（包括机动操纵）时，驾驶室控制站应能对主推进装置的推进方向和推进速度及其他有关装置（如设有时）实施有效的控制，其设计和布置应满足本篇第2章第8节的要求。

4.3.2.2 必要时，驾驶室控制站对推进装置的控制应能在任何时候转换到就地控制站，且控制转换应满足本篇2.5.2的有关要求。

4.3.2.3 对主机、离合器、调距桨等的自动控制和遥控功能，一般应满足本篇第3章第2节的有关要求。

4.3.2.4 主机的a类保护动作应能自动进行，其余的保护动作（b类和c类）可以由人工进行操作。

4.3.2.5 驾驶室控制站的报警：

（1）驾驶室控制站报警的项目和报警方式见本章表4.3.4.1第5栏，被监控的机电设备和自动化监控系统的故障报警一般可以4种方式延伸到驾驶室控制站：安全系统动作的组合报警、单独报警、严重故障的组合报警和一般故障的组合报警；

（2）在驾驶室控制站的听觉报警信号允许在报警被应答后消声，但视觉报警信号应在故障排除后消除，而且，视觉报警信号应能在故障报警被应答前后有所区别。

4.3.2.6 如主机由驾驶室控制站遥控起动时，驾驶室控制站应显示主机起动空气压力或起动蓄电池组电压。

# 4.3.3 就地控制站的功能

4.3.3.1 就地控制站应满足本篇2.5.1.5的要求。

4.3.3.2 就地控制站应对推进装置（如主机、离合器、调距桨等）设置表 4.3.4.1 第 2 栏和第 3 栏中要求显示、报警的项目。
