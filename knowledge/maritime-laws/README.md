# 海事法规知识库管理系统

基于 **三级索引 + 章节级精准加载** 的法规管理系统，专为大量PDF转换的md文件设计，支持高效跨项目引用和合规评审。

## 🎯 设计目标

1. **Token节约**：启动时仅加载索引（~3K tokens），按需加载具体章节
2. **精准引用**：支持章节级引用，避免全文加载
3. **人类友好**：一键处理，中文界面，简单维护
4. **评审优化**：为法规政策评审设计，快速定位相关条款

## 📁 系统结构

```
maritime-laws/
├── raw/                    # 原始md文件（用户放入PDF转换的文件）
├── processed/             # 处理后的文件
│   ├── chapters/         # 按章节拆分的文件（按需加载）
│   └── topic-index.md    # 主题交叉索引
├── indices/              # 索引文件
│   ├── file-indices/    # 单个文件的详细索引
│   ├── maritime-index.md # 海事法规总索引（核心文件）
│   └── update-report.txt # 更新报告
└── tools/                # 处理工具
    ├── process-maritime-laws.sh # 一键处理脚本
    └── analyze-md.py     # 文件分析工具
```

## 🚀 快速开始

### 步骤1：准备原始文件
1. 将PDF转换为markdown文件（推荐使用mineru等工具）
2. 确保转换质量良好，章节标题清晰
3. 将md文件放入 `maritime-laws/raw/` 目录

### 步骤2：一键处理
```bash
cd /Users/fanxiaojun/Desktop/Workspace/knowledge_知识库/maritime-laws/tools
./process-maritime-laws.sh
```

**脚本功能**：
- ✅ 自动分析文件结构（章节、元数据、关键词）
- ✅ 生成详细文件索引
- ✅ 按章节拆分大文件（可选）
- ✅ 更新海事法规总索引
- ✅ 生成更新报告和统计

### 步骤3：更新知识库主索引
编辑 `knowledge_知识库/knowledge-index.md`，添加海事法规分类：

```markdown
### 4. 海事法规 (maritime-laws/)
IMO公约、国内海事法规、技术标准等，共 X 个文件。

| 文件 | 发布机构 | 生效日期 | 关键词 | 章节数 | 简要描述 |
|------|----------|----------|--------|--------|----------|
| `SOLAS公约.md` | IMO | 1974-05-25 | 安全, 公约, 构造 | 14 | 国际海上人命安全公约 |
```

**提示**：脚本会生成 `update-report.txt` 提供具体更新内容。

## 📚 使用指南

### 1. 跨项目引用格式
在项目文档中使用统一引用格式：

```markdown
[海事法规：SOLAS公约-Chapter-II-2] - 引用SOLAS公约第II-2章（防火）
[海事法规：MARPOL-Annex-VI] - 引用MARPOL公约附则VI（空气污染）
```

**优势**：
- Claude只需加载 `maritime-index.md`（~3K tokens）了解全貌
- 按需加载具体章节（~1-2K tokens），而非整个文件（~50K+ tokens）
- 相同章节跨项目复用，避免重复学习

### 2. 合规评审工作流
1. **启动项目**：阅读 `maritime-index.md` 了解相关法规
2. **关键词检索**：通过关键词索引快速定位条款
3. **精准加载**：仅加载相关章节文件
4. **对比分析**：使用章节文件进行条款对比
5. **生成意见**：基于具体条款提出评审意见

### 3. 日常维护
- **新增法规**：放入 `raw/` 目录，重新运行脚本
- **更新法规**：替换 `raw/` 中的文件，重新运行脚本
- **定期检查**：每月运行一次脚本，更新索引统计

**统一纳管规则**：
- 法规原文统一纳入 `raw/`，`maritime-laws/` 根目录和 `processed/` 根目录不保留正式法规原文
- 正式章节切片统一纳入 `processed/chapters/`
- 正式单文件索引统一纳入 `indices/file-indices/`
- 主题交叉索引维护在 `processed/topic-index.md`

## 🔧 技术特性

### 章节级精准加载
- 200页PDF → 拆分为20-30个章节文件
- 每个章节文件 0.5-2K tokens
- 查找具体条款时加载成本降低 **95%+**

### 智能分析
- 自动提取章节结构（支持6级标题）
- 元数据识别（发布机构、生效日期等）
- 关键词自动提取
- 智能分类（国际公约/国内法规/技术规范）

### 渐进式处理
- 支持Python3分析（推荐）或简化处理
- 章节拆分可选，可按需启用
- 出错时自动降级到简化模式

## 📊 性能对比

## 💡 最佳实践

### 文件命名建议
- 包含法规名称和年份：`SOLAS公约-2024修正.md`
- 使用中文名称便于识别
- 避免特殊字符

### 转换质量要求
- 确保章节标题正确识别（#、##、###）
- 保留元数据信息（发布机构、生效日期）
- 转换后检查关键条款完整性

### 评审专用技巧
1. **建立评审模板**：在项目模板中预置法规引用格式
2. **关键词标准化**：统一使用"防火"而非"火灾防护"等
3. **版本记录**：在文件索引中记录法规修订历史
4. **影响分析**：跟踪新法规对现有项目的影响

## 🛠️ 故障排除

### 常见问题
1. **Python分析失败**：确保系统有Python3，或使用简化模式
2. **章节识别错误**：检查md文件标题格式是否正确
3. **分类错误**：手动调整 `update_main_index` 函数中的分类逻辑
4. **索引更新失败**：查看 `update-report.txt` 手动更新

### 日志文件
- `maritime-index.md.backup.*` - 索引备份
- `update-report.txt` - 更新报告
- `tools/analyze-md.py` 输出 - 分析日志
